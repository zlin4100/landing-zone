## 结构说明：宏观数据-核心 vs 量化建模数据（研究版口径）

为避免“同一指标既做信号又做收益腿”导致解释混乱，本清单按用途分为两条主线：

1. **宏观数据-核心（体制识别 / 象限识别 / 风控状态）**  
  - 用于判别：增长/通胀/流动性/信用等 regime  
  - 允许：日频信号 → 月度聚合用于建模（但应保留日频原始以便审计/复算）  
  - 不直接承担“可交易收益腿”的可复现对标（除非明确说明合成口径）
2. **量化建模数据（收益腿构建 / 回测 / 风险预算与再平衡）**  
  - 直接用于构造：CASH/BOND/EQUITY/COMMODITY 等收益序列与协方差矩阵  
  - 必须：口径稳定、回测可复算（特别是期货连续合约、杠杆资金成本、交易成本）

> 全天候（All Weather）落地的最小关键补齐：**超长久期利率（30Y）+ 通胀预期（T5YIE）**，并配套 **DR007 日频资金成本** 与期限利差等内部特征（银行可落地版本）。

### 1.x 全天候策略补充：内部合成特征（非采购项）

以下指标 **不新增采购成本**，由已采购/已接入的原始序列在系统内计算生成，用于全天候（四象限）体制识别与可复算披露：

> **内部合成说明**（非采购项）:
>
> - COMMODITY 由系统按 `r_commodity = 0.7 × r_黄金ETF + 0.3 × r_南华商品` 合成
> - BOND 月度收益按久期近似法计算：`r_bond = (y/12) - D × Δy`（10Y 示例 D≈7；30Y 可取更长久期，必要时可加入凸性项以提高极端行情拟合）
> - CASH 月度收益按 `r_cash = (SHIBOR_rate / 100) / 12` 计算
> - 详见 `reports/OPTIMIZATION_RESULTS.md`

> **历史窗口说明**:
>
> - 原窗口为近3年（2023-01 至今），权益类资产均值估计标准误过大（年波动率20%时，SE = 20%/sqrt(3) = 11.5%），估计几乎无统计意义
> - 扩展至近20年后（若可得；SE ≈ 20%/sqrt(20) = 4.5%），预期收益率均值估计精度相对10年样本再提升约29%；同时保留近10年窗口用于稳健性对照
> - 四资产“可回测交集窗口”仍受限于权益/黄金ETF上市日期（2012/2013），但原始腿建议尽量采购到近20年，以便：①利率/资金成本腿的长周期估计；②后续替换为更长历史的指数/基金口径后可直接复用
