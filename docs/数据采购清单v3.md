## 三、分类调整后的理解要点

### 1. 本次重分类的核心变化

- 不再沿用“宏观周期与政策环境 / 市场动能拆解 / 国际环境 / 大类资产”这种偏业务流程的章节命名。
- 改为更接近《宏观数据跟踪框架》和《指标表格》的数据研究分类：
  - 经济总量与景气度
  - 价格与通胀
  - 消费
  - 投资与房地产
  - 进出口与外需
  - 货币与信用
  - 财政与政策工具
  - 国际环境与风险偏好
  - 大类资产建模数据

### 2. 这样分类的好处

- 更接近卖方宏观、资产配置、数据治理的通用口径。
- 更利于后续做指标检索、数据分层、权限管理和看板展示。
- 业务解读时仍可按“宏观 → 动能 → 国际 → 资产”链路输出，但底层数据目录不再混杂业务逻辑与指标分类逻辑。

### 3. 重复归类的处理原则

以下指标天然具备跨分类属性：

- `PMI_NEW_EXPORT_ORDERS`：既属于景气分项，也属于出口前瞻。
- `BRENT_CRUDE`：既属于价格外生冲击，也属于国际环境变量。
- `LOCAL_SPECIAL_BOND_TARGET_ANNUAL`：既属于投资锚点，也属于财政政策工具。
- `CGB_3Y`、`AA_CREDIT_YIELD_3Y`：既属于固收建模，也属于货币信用分析。

处理原则：

- **底层只保留一套原始数据；上层目录可按用途多入口映射。**

## 四、内部合成特征（非采购项）

以下指标由已采购原始序列在系统内计算，不新增采购成本：

- `TERM_SPREAD_10Y_DR007 = CGB_10Y - DR007`
- `CPI_PPI_SCISSOR = CPI_YOY - PPI_YOY`
- `DR007_FUNDING_COST_DAILY`：按 DR007 日频逐日计提资金成本
- `CREDIT_SPREAD_3Y_AA = AA_CREDIT_YIELD_3Y - CGB_3Y`
- `r_cash_A ≈ MMF_7D_YIELD_M / 12`
- `r_bond_factor ≈ -duration × ΔCGB_10Y`
- `r_equity_A = ln(CSI300_TR_t / CSI300_TR_{t-1})`
- `r_equity_B = ln(CSI300_t / CSI300_{t-1})`

## 五、一句话总结

这版 v3 的重点不是删减指标，而是把原本按“业务分析流程”组织的采购清单，重排成一套更贴近**宏观研究框架 + 指标库管理框架**的分类方式：

> **上层按研究框架分类，底层按原始指标去重，业务侧再按市场分析链路调用。**
