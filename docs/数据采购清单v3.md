## 三、分类调整后的理解要点

### 1. 本次重分类的核心变化

- 不再沿用“宏观周期与政策环境 / 市场动能拆解 / 国际环境 / 大类资产”这种偏业务流程的章节命名。
- 改为更接近《宏观数据跟踪框架》和《指标表格》的数据研究分类，并要求每个原始指标只保留一个归属分类：
  - 经济总量与景气度
  - 价格与通胀
  - 消费
  - 投资与房地产
  - 进出口与外需
  - 货币与信用
  - 财政与政策工具
  - 国际环境与风险偏好
  - 大类资产建模数据

### 2. 这样分类的好处

- 更接近卖方宏观、资产配置、数据治理的通用口径。
- 更利于后续做指标检索、数据分层、权限管理和看板展示。
- 业务解读时仍可按“宏观 → 动能 → 国际 → 资产”链路输出，但底层数据目录不再混杂业务逻辑与指标分类逻辑；同时每个原始指标只保留一个归属分类，避免重复交叉。

### 3. 分类处理原则

处理原则：

- **每个原始指标只归属一个一级分类，不做重复交叉。**
- **若某指标兼具多种解释用途，则按“原始经济含义优先”确定其主归属。**
- **大类资产建模数据章节只保留上文未出现、但建模额外需要的指标。**

## 四、内部合成特征（非采购项）

以下指标由已采购原始序列在系统内计算，不新增采购成本：

- `TERM_SPREAD_10Y_DR007 = CGB_10Y - DR007`

- `r_cash_A ≈ MMF_7D_YIELD_M / 12`

- `r_equity_A = ln(CSI300_TR_t / CSI300_TR_{t-1})`
- `r_equity_B = ln(CSI300_t / CSI300_{t-1})`

## 五、一句话总结

这版 v3 的重点不是删减指标，而是把原本按“业务分析流程”组织的采购清单，重排成一套更贴近**宏观研究框架 + 指标库管理框架**的分类方式：

> **上层按研究框架分类，底层按原始指标去重，业务侧再按市场分析链路调用。**
