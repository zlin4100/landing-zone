# **一、核心目标（一句话）**

> **将东方财富快讯中的“新闻”，转化为可用于资产分析的“原子事件”，并在周频层面形成稳定的市场主线。**

---

# **二、什么是“原子事件”（核心定义）**

## **定义**

> **原子事件 = 不包含推理、只描述“已发生变化”的最小市场事实单元**

---

## **必须满足 3 个条件：**

### **1）可验证（必须有证据）**

- 有明确主体（谁）
- 有明确动作（做了什么）
- 有明确结果（发生了什么变化）

---

### **2）单跳因果（不能推演）**

只能到：

```
事件 → 直接影响
```

不能扩展到：

```
事件 → 通胀 → 利率 → 股票
```

---

### **3）可映射资产**

必须至少能映射到一类资产：

- 股票类
- 固收类
- 另类资产
- 现金类

---

## **❌ 非原子事件（典型错误）**

```
市场情绪回暖
经济预期改善
风险偏好提升
```

👉 没有“动作”，全部是解释

---

## **✔ 原子事件（正确）**

```
央行开展逆回购
原油价格上涨
债券收益率下行
```

👉 都是“物理变化”

---

# **三、市场事件选择标准（P0筛选）**

从日频抓到100条新闻，只保留极少数进入系统。

---

## **P0筛选规则（必须同时满足）**

### **1）影响范围**

- ✔ 全市场 / 大类资产
- ❌ 个股 / 单行业

---

### **2）价格验证**

- ✔ 有市场反应（价格/利率/数据）
- ❌ 纯政策表述（无反馈）

---

### **3）机制清晰**

- ✔ 能明确“作用路径”（流动性/供给/信用等）
- ❌ 无法解释

---

### **4）非重复**

- ✔ 与已有事件不同
- ❌ 同一事件不同表述

---

# **四、处理机制（系统流程）**

## **总体流程（最终版）**

```
新闻
→ LLM提取原子事件
→ 规则校验（不过就丢）
→ LLM抽象机制（单跳）
→ 日频事件库
→ 周频主线聚合
```

---

## **Step 1：原子事件提取（LLM + 规则校验）**

### **Step 1.1：LLM提取（宽松）**

从新闻中提取：

```
{
  "subject": "...",
  "action": "...",
  "object": "...",
  "result": "..."
}
```

要求：
- 只提取最核心事件
- 不解释、不推理
- 保持简洁

---

### **Step 1.2：规则校验（强约束）**

只保留满足以下条件的事件：

**结构完整：**
- subject 不为空
- action 不为空
- result 不为空

**动作合法（白名单）：**
```
封锁 / 投放 / 回笼 / 上调 / 下调 / 增加 / 减少 / 违约 / 扩大 / 收紧 / 放松
```

**结果必须是“变化”：**
```
上涨 / 下跌 / 受阻 / 净投放 / 净回笼 / 收紧 / 放松
```

👉 不满足条件：
```
丢弃（不进入系统）
```

---

## **Step 2：机制抽象（受限LLM）**

将原子事件转为：

```
{
  "event": "...",
  "event_type": "...",
  "transmission_path": ["单一机制"],
  "asset_class": [...],
  "impact_direction": "...",
  "market_validation": "..."
}
```

---

## **⚠️ 强约束（必须执行）**

```
1. transmission_path 只能1个（最多2个紧邻）
2. 禁止跨变量推理（通胀/利率等）
3. impact_direction 必须有市场数据支持
4. 不允许出现“预期/可能/或将”
```

---

## **设计原则（极简版）**

```
Step 1：让模型提取事实（但必须通过规则过滤）
Step 2：让模型做最小抽象（单跳机制）
```

```
模型负责“覆盖复杂性”
规则负责“保证确定性”
```

# **六、周频输出（给投顾使用）**

每周输出：

```
每类资产：
- 1~3个主线事件
- 每个事件必须：
  - 单跳机制
  - 市场验证
  - 明确方向
```

# **七、完整案例（你要求的两个）**

---

# **案例1：霍尔木兹海峡封锁**

## **🧾 原始新闻（真实）**

- 伊朗封锁霍尔木兹海峡
- 石油运输受阻
- 原油价格上涨

---

## **Step 1：原子事件**

```
{
  "subject": "伊朗",
  "action": "封锁海峡",
  "object": "霍尔木兹海峡",
  "result": "石油运输受阻 + 原油价格上涨"
}
```

---

## **Step 2：机制事件（合规版本）**

```
{
  "event": "原油运输受阻导致供给紧张",
  "event_type": "地缘政治",
  "asset_class": ["另类资产"],
  "transmission_path": ["供给"],
  "impact_direction": {
    "原油": "上涨"
  },
  "market_validation": {
    "原油价格": "上涨"
  },
  "key_signal": [
    "海峡封锁",
    "运输受阻",
    "油价上涨"
  ]
}
```

---

## **❗ 注意（关键）**

没有写：

```
通胀 ↑
利率 ↑
股票 ↓
```

👉 这些全部交给模型推理层

# **八、最终一句话总结（系统原则）**

> **原子事件只描述“发生了什么变化”，绝不描述“未来会发生什么”。**

---

# **九、你这个模块的本质（非常重要）**

你现在构建的不是：

> 新闻系统

而是：

> **市场状态变化的最小表达系统**

---

# **十、最终架构（可以直接写PRD）**

```
日频：
东方财富快讯 → 原子事件 → 单跳机制事件 → 入库

周频：
事件聚合 → 主线提取 → 四大类资产分析输出
```

---

这套方案的优点只有三个，但很致命：

- 不依赖观点（抗偏见）
- 不过度推理（抗幻觉）
- 可稳定复用（工程可落地）
