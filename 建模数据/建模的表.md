## 数学建模核心数据库表（单表方案）

刚才的理解需要修正：这里不是 6 张表，而是 **1 张数学建模核心表中的 5 个核心变量**。

也就是说，数据库层更适合先落成一张总表，再把下面 5 类内容作为表内字段组或逻辑字段域统一管理。

### 这 1 张表中的 5 个核心变量

#### 1. 大类资产年化波动率 `asset_volatility_sigma`
- 含义：四大类资产各自的年化波动率，用于描述单资产风险强弱
- 适合承载的内容：
  - 资产类别
  - 观察窗口（1Y / 3Y / 5Y / 10Y / 20Y）
  - 年化波动率
  - 计算口径
  - 生效日期 / 截止日期

#### 2. 大类资产年化协方差矩阵 `asset_covariance_matrix_sigma`
- 含义：用于组合风险计算，是资产配置求解的核心输入之一
- 适合承载的内容：
  - 行资产类别
  - 列资产类别
  - 观察窗口（1Y / 3Y / 5Y / 10Y / 20Y）
  - 协方差值
  - 年化标识
  - 估计方法 / 计算日期

#### 3. 大类资产长期相关性矩阵 `asset_correlation_matrix_rho`
- 含义：描述四大类资产之间的联动结构，更偏解释、校验和风格识别
- 适合承载的内容：
  - 行资产类别
  - 列资产类别
  - 观察窗口（1Y / 3Y / 5Y / 10Y / 20Y）
  - 相关系数
  - 计算口径
  - 截止日期

#### 4. 无风险利率 `risk_free_rate_rf`
- 含义：提供建模无风险利率基准，以及长短端利率结构信息
- 你当前清单里已经有比较清晰的底层支撑：
  - `CGB_1Y`：建模无风险利率
  - `CGB_10Y`：长端无风险利率锚
  - 内部合成特征里还定义了 `r_f = CGB_1Y` 和 `TERM_SPREAD_10Y_1Y = CGB_10Y - CGB_1Y`
- 适合承载的内容：
  - 期限（1Y / 3Y / 10Y）
  - 利率值
  - 基准类型
  - 日期
  - 是否为建模主用利率

#### 5. 四大类资产基准画像 `asset_benchmark_profile`
- 含义：沉淀四大类资产在不同窗口下的收益、波动、Sharpe 等画像数据
- 这里的 Benchmark 更适合理解为一组基准特征，而不是单一指数名称
- 适合承载的内容：
  - 资产类别
  - 基准指数名称
  - `ann_return_1y / 3y / 5y / 10y / 20y`
  - `ann_vol_1y / 3y / 5y / 10y / 20y`
  - `sharpe_ratio_1y / 3y / 5y / 10y / 20y`
  - 截止日期

## 一句话理解这 5 个变量

这不是 5 张表，而是 **1 张数学建模核心表里的 5 类关键变量**：

- 风险变量：`asset_volatility_sigma`、`asset_covariance_matrix_sigma`、`asset_correlation_matrix_rho`
- 利率变量：`risk_free_rate_rf`
- 基准画像变量：`asset_benchmark_profile`

## 最终推荐口径

你后面无论是写数据库、接口、PRD，还是喂给大模型，都可以统一成这版：

**一张数学建模核心表，包含 5 个核心变量：**
1. 大类资产年化波动率 `asset_volatility_sigma`
2. 大类资产年化协方差矩阵 `asset_covariance_matrix_sigma`
3. 大类资产长期相关性矩阵 `asset_correlation_matrix_rho`
4. 无风险利率 `risk_free_rate_rf`
5. 四大类资产基准画像 `asset_benchmark_profile`

### 5 个核心变量的英文变量名

| 中文名称 | 数学对象 | 英文变量名 | 说明 |
|---|---|---|---|
| 大类资产年化波动率 | `σ` | `asset_volatility_sigma` | 四大类资产单资产风险输入 |
| 大类资产年化协方差矩阵 | `Σ` | `asset_covariance_matrix_sigma` | 组合风险计算核心输入 |
| 大类资产长期相关性矩阵 | `ρ` | `asset_correlation_matrix_rho` | 资产联动关系与风险结构解释 |
| 四大类资产基准画像 | `Benchmark` | `asset_benchmark_profile` | 四大类资产收益、波动、Sharpe 画像 |
| 无风险利率 | `r_f` | `risk_free_rate_rf` | Sharpe 与超额收益基准 |

这版的优点是：
- 更符合“单表承载建模核心输入”的理解
- 和数学建模对象一一对应
- 和你当前采购清单里的利率、资产腿、内部合成特征也能自然衔接
- 避免 `asset_benchmark_profile` 与 `asset_expected_return_mu` 在长期画像口径上的重叠

下一步最适合做的，就是把这 **1 张表** 直接展开成 **“表名 + 主键 + 字段清单 + 字段中文解释”**。
