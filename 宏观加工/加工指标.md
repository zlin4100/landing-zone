# 单独汇总：需加工的原始指标 → 加工方式 → 最终喂给模型指标

这一节只单独汇总两类：

1. 原始日度指标，需先转成月度后再进入模型输入。
2. 专项债相关指标，需先基于原始指标加工派生后，再作为最终输入指标。

### 1）日度转月度：原始指标 → 月频口径 → 原因 → 最终喂给模型指标

| 类型    | 原始指标代码             | 原始指标名称            | 月频口径 | 原因 | 是否改名 | 最终喂给模型指标         |
| ----- | ------------------ | ----------------- | ---- | ---- | ---- | ---------------- |
| 日度转月度 | BRENT_CRUDE        | 期货收盘价(连续):ICE布油   | 月均值 | 最近波动较大，统一用月均值可降低单日尖刺和月末事件冲击对模型输入的干扰，更适合月度宏观判断。 | 否    | BRENT_CRUDE      |
| 日度转月度 | XAUUSD             | 期货收盘价(连续):COMEX黄金 | 月均值 | 最近波动较大，统一用月均值可降低单日尖刺和月末事件冲击对模型输入的干扰，更适合月度宏观判断。 | 否    | XAUUSD           |
| 日度转月度 | DR007              | DR007             | 月均值 | 属于短端资金利率，受月末、季末、节假日和监管考核扰动较大，月均值更能代表当月整体资金面松紧。 | 否    | DR007            |
| 日度转月度 | FX_CNY_MID         | 中间价:美元兑人民币       | 月均值 | 属于日度汇率中间价，受单日市场波动、月末时点因素和短期外部冲击影响较大，月均值更能代表当月整体汇率环境。 | 否    | FX_CNY_MID       |

- 说明：
 
- 这几类指标（包括原油、黄金、汇率、波动率指数、资金利率和债券收益率）都不能直接把原始日频序列喂给模型，而应先统一整理成月频。
- 当前统一采用月均值，目的是降低单日异常波动、月末时点噪声和突发事件冲击对模型判断的干扰。
- 对模型来说，月均值表示“该月整体环境大致处于什么水平”，比单一月末值更稳定，更适合做月度宏观与市场状态输入。

### 2）专项债加工：原始指标 → 加工方式 → 最终喂给模型指标


| 类型    | 原始指标代码                                                    | 原始指标名称                    | 必要加工                                                        | 是否改名 | 最终喂给模型指标              | 备注          |
| ----- | --------------------------------------------------------- | ------------------------- | ----------------------------------------------------------- | ---- | --------------------- | ----------- |
| 专项债加工 | SPECIAL_BOND_ISSUE_CUM + LOCAL_SPECIAL_BOND_TARGET_ANNUAL | 新增专项债券:累计值 + 政府预期目标:地方专项债 | `SPECIAL_BOND_ISSUE_CUM / LOCAL_SPECIAL_BOND_TARGET_ANNUAL` | 是    | SPECIAL_BOND_PROGRESS | 适合作为最终月度主指标 |


补充说明：

- `SPECIAL_BOND_ISSUE_CUM` 本身是原始累计值，但保留它有意义，因为它能补充专项债发行的绝对量级信息。
- `SPECIAL_BOND_PROGRESS` 是基于 `SPECIAL_BOND_ISSUE_CUM` 和 `LOCAL_SPECIAL_BOND_TARGET_ANNUAL` 加工得到的最终指标，用于表示年度执行进度。
- 喂给模型时，不需要在每一行额外强调“月均值”或“月末值”，因为月频口径应在指标定义文档中一次性约定清楚；模型输入表保持统一结构即可。

### 3）喂给模型的建议

推荐直接保持统一结构，不要在每一行 value 里重复写“月均值”或“月末值”，而是在指标定义层提前约定口径。

推荐格式：

```csv
indicator_id,ref_month,value,unit,adjustment,source,data_type
LPR_1Y,2024-11,3.1,%,Level,PBOC,Rate
LPR_1Y,2024-12,3.1,%,Level,PBOC,Rate
```

说明：

- `indicator_id` 表示指标本身，不额外把 `MONTH_AVG` 或 `MONTH_END` 拼进代码里。
- `ref_month` 表示该值对应的月份。
- `value` 直接放最终月度值。
- 月频口径（例如“月均值”）应写在单独的指标定义文档中，而不是重复塞进模型输入表。
- 只有在同一个指标同时存在多种月频口径并且确实要同时喂给模型时，才需要额外新增字段区分。
