喂给模型例子：

```csv
indicator_id,ref_month,value,unit,adjustment,source,data_type
CPI_YOY,2024-01,-0.8,%,YoY,NBS,Inflation
CPI_YOY,2024-02,0,%,YoY,NBS,Inflation
CPI_YOY,2024-03,0,%,YoY,NBS,Inflation
```

A. 可直接月频喂原值

这类原值本身就有较强解释性，可直接喂给模型：

- 年度同比类指标：原值已经表达方向与强弱，可直接喂。
- 累计同比类指标：原值已经表达方向与强弱，可直接喂。
- 50 上下阈值类指标：level 本身就有明确经济含义，可直接喂。
- 指数类 level 指标：原值本身已有较强经济解释性，可直接喂。

建议直接喂原值的指标：

- GDP_REAL_YOY（同比原值可直接喂；但它是季频，仅季末月作为低频确认项）
- M1_YOY
- CONSUMER_CONFIDENCE
- PMI_NEW_EXPORT_ORDERS
- EXPORT_AMOUNT_YOY
- IMPORT_AMOUNT_YOY
- INFRA_INVEST_CUM_YOY
- MANUFACTURING_INVEST_CUM_YOY
- REAL_ESTATE_INVEST_CUM_YOY

B. 需月度化

这类指标不建议直接把原始高频数据喂给模型，而应先统一到月频：

- 日频价格类：先转月均值、月末值或月度收益率。
- 日频利率类：先转月均值或月末值。

建议先月度化再喂的指标：

- BRENT_CRUDE
- XAUUSD
- DR007
- AA_CREDIT_YIELD_3Y
- CGB_3Y

C. 需派生后再喂

这类指标原始值不适合直接作为月度宏观判断输入，更适合先做派生特征：

- 累计值类：优先转成当月新增、同比或执行进度。
- 年度目标类：更适合作为分母或年度政策锚，而不是直接当月频动态变量。

最后喂给模型时，专项债相关指标建议优先保留：

- SPECIAL_BOND_PROGRESS
- SPECIAL_BOND_ISSUE_CUM

说明：

- SPECIAL_BOND_PROGRESS：用于表示当年专项债发行执行到什么程度，是最核心的相对进度指标。
- SPECIAL_BOND_ISSUE_CUM：用于表示当前累计发行规模，补充绝对量级信息。
- SPECIAL_BOND_ISSUE_MONTHLY：不再作为最终喂给模型的指标。
- SPECIAL_BOND_ISSUE_CUM_YOY：与执行进度存在较强重叠，不作为最终喂给模型的 P0 / P1 主指标。

对应派生关系：

- SPECIAL_BOND_PROGRESS = SPECIAL_BOND_ISSUE_CUM / LOCAL_SPECIAL_BOND_TARGET_ANNUAL

## D. 需要逆向的原始指标

这里仅针对本 markdown 中出现的原始指标，不包含派生后的指标。
这一节只回答一个问题：**哪些原始指标还需要逆向处理**。
不在这里讨论 `indicator_id` 应该怎么命名，也不写任何“建议 indicator_id”。

### 判断原则

以下几类原始指标，通常仍然需要逆向处理后再进入最终给模型的输入：

- 供应商代码感太强，单看代码无法直接理解含义。
- 原始 code 过短，语义不足。
- 原始指标名和最终喂给模型的表达之间，还隔着一层“业务解释”。
- 原始数据如果是日度、累计值、年度目标值，通常不能直接作为月度判断输入。

## E. 单独汇总：需加工的原始指标 → 加工方式 → 最终喂给模型指标

这一节只单独汇总两类：

1. 原始日度指标，需先转成月度后再进入模型输入。
2. 专项债相关指标，需先基于原始指标加工派生后，再作为最终输入指标。

### 1）日度转月度：原始指标 → 加工方式 → 最终喂给模型指标


| 类型    | 原始指标代码             | 原始指标名称            | 必要加工            | 是否改名 | 最终喂给模型指标         |
| ----- | ------------------ | ----------------- | --------------- | ---- | ---------------- |
| 日度转月度 | BRENT_CRUDE        | 期货收盘价(连续):ICE布油   | 日度 → 月度         | 否    | BRENT_CRUDE      |
| 日度转月度 | XAUUSD             | 期货收盘价(连续):COMEX黄金 | 日度 → 月度         | 否    | XAUUSD           |
| 日度转月度 | DR007              | DR007             | 日度 → 月度         | 否    | DR007            |
| 日度转月度 | AA_CREDIT_YIELD_3Y | AA级信用债收益率（3年）     | 日度 → 月度 + 命名规范化 | 是    | CREDIT_YTM_AA_3Y |
| 日度转月度 | CGB_3Y             | 中债国债到期收益率:3年      | 日度 → 月度 + 命名规范化 | 是    | CGB_3Y_YTM       |


说明：

- 这几类指标都不能直接把原始日频序列喂给模型，而应先统一整理成月频。
- 其中一部分指标仅做频率转换，不改最终 `indicator_id`；另一部分除了月度化，还同时做了命名逆向规范化。

### 2）专项债加工：原始指标 → 加工方式 → 最终喂给模型指标

| 类型 | 原始指标代码 | 原始指标名称 | 必要加工 | 是否改名 | 最终喂给模型指标 | 备注 |
| --- | --- | --- | --- | --- | --- | --- |
| 专项债加工 | SPECIAL_BOND_ISSUE_CUM | 新增专项债券:累计值 | 保留为原始累计值，直接作为最终输入之一 | 否 | SPECIAL_BOND_ISSUE_CUM | 用于补充绝对规模信息 |
| 专项债加工 | SPECIAL_BOND_ISSUE_CUM + LOCAL_SPECIAL_BOND_TARGET_ANNUAL | 新增专项债券:累计值 + 政府预期目标:地方专项债 | `SPECIAL_BOND_ISSUE_CUM / LOCAL_SPECIAL_BOND_TARGET_ANNUAL` | 是 | SPECIAL_BOND_PROGRESS | 适合作为最终月度主指标 |


补充说明：

- `SPECIAL_BOND_ISSUE_CUM` 本身是原始累计值，但保留它有意义，因为它能补充专项债发行的绝对量级信息。
- `SPECIAL_BOND_PROGRESS` 是基于 `SPECIAL_BOND_ISSUE_CUM` 和 `LOCAL_SPECIAL_BOND_TARGET_ANNUAL` 加工得到的最终指标，用于表示年度执行进度。
- `SPECIAL_BOND_ISSUE_MONTHLY` 不再作为最终喂给模型的指标。
